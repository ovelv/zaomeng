# AI 萌宠视频生成器 产品设计 (PRD)

## 1. 核心目标
帮助用户通过一句“内容主题”，自动生成分镜脚本，并通过固定角色形象、图生图和局部重绘的思路，批量生成多镜头叙事、画面绝对一致的萌宠短视频。

## 2. 业务流程 (3 个核心步骤)

### Step 1: 脚本与分镜生成
- **输入**: 用户输入“内容主题”（如：一只戴着红围巾的金毛在魔法森林里寻找宝藏）。
- **处理**: 调用 `Gemini-3.1-Pro-Preview`，基于大模型强大的叙事能力，自动拆解出 3-5 个分镜。
- **输出**: 每个分镜包含：
  - `scene`: 场景序号
  - `description`: 画面主体描述（用于生图）
  - `camera`: 镜头语言与运镜（用于生视频）
  - `duration`: 预估时长（秒）

### Step 2: 角色底图生成 (图生图 & 垫图)
- **输入**: 全局角色设定（如：一只戴着红围巾的金毛）、用户上传/选择的全局垫图（参考图 URL 或 assetId）。
- **处理**: 
  - 为每个分镜生成一张“底图”。
  - 核心诉求是**“长得绝对一致”**。这在工程上通过调用支持 `image-to-image` (图生图) 或带有 IP-Adapter / 局部重绘能力的生图模型（如火山引擎 Seedream 或同类文生图 API）实现。
  - 将“全局角色设定”与“分镜画面描述”拼接，同时将“全局垫图”作为参考图传入模型。
- **输出**: 每个分镜获得一张固定形象的静态底图。允许用户单独重新生成某一张不满意的底图。

### Step 3: 多镜头叙事视频生成
- **输入**: 每个分镜生成的“底图”、分镜的“运镜与提示词”、视频生成参数。
- **处理**: 
  - 遍历所有分镜，调用 `Seedance 2.0`。
  - 使用图生视频（首帧参考），将第二步生成的底图作为 `first_frame` 或 `reference_image` 传入 Seedance。
  - 附加运镜提示词，生成连续的叙事视频。
- **输出**: 多段基于同一主角的动态视频，最终可供用户下载并剪辑成完整的萌宠故事。

## 3. UI 界面设计

- **全局导航**: 顶部或侧边栏提供 `口播视频` 与 `萌宠视频` 切换。
- **萌宠生成器页面布局**:
  - **模块 A: 全局设定**: 
    - 内容主题输入框
    - 全局角色描述输入框
    - 主角垫图 (图片 URL 或 assetId)
    - 按钮：`1. 生成分镜脚本`
  - **模块 B: 分镜与底图生成**: 
    - 列表展示每个分镜（支持手动编辑提示词和运镜）。
    - 按钮：`2. 批量生成分镜底图`
    - 图片位：展示生图结果，单张重绘按钮。
  - **模块 C: 视频生成**: 
    - 设置：视频比例、默认单镜时长、模型选择。
    - 按钮：`3. 启动多镜头视频生成`
    - 结果展示：同现有的 Seedance 任务列表，展示视频状态与预览。

## 4. 技术架构与 API

1. **Gemini 路由** (`/api/gemini/storyboard`): 封装 `@google/genai`，输出严格的 JSON Array。
2. **生图路由** (`/api/image/generate`): 封装火山引擎文生图/图生图 API (如 Seedream) 或占位实现，接收 `prompt` 和 `reference_image`，返回图片 URL。
3. **视频路由** (`/api/jobs/pets` 或复用现有): 接收图片 URL 作为首帧，调用 `Seedance 2.0`。
