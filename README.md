# 造梦口播视频生成器 V1

一个基于 Next.js 的 V1 原型：

## 核心功能

### 1. 连续口播视频生成器
- 输入长口播文案，自动按字数切分为短视频脚本。
- 每段固定使用 `assetId`、固定人设模板和固定口播镜头模板生成视频。
- 每段视频时长可单独控制（默认 4-15 秒）。

### 2. AI 萌宠视频生成器
- 输入故事主题，调用 `Gemini-3.1-Pro-Preview` 自动生成分镜脚本。
- 基于全局角色设定和垫图（支持上传的垫图 URL 或 assetId），批量生成各个分镜的角色底图（使用图生图或 Seedream，保持长相绝对一致）。
- 串联 `Seedance 2.0` 图生视频能力，配合各分镜的运镜提示词，完成多镜头叙事。

## 本地启动

1. 安装依赖

```bash
npm install
```

2. 配置环境变量

```bash
cp .env.example .env.local
```

至少填写：

```bash
ARK_API_KEY=你的方舟 API Key
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
SEEDANCE_MODEL=doubao-seedance-2-0-260128
```

3. 启动开发环境

```bash
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000)

## 当前实现说明

- 文案切分：按 60-70 字目标切分，优先在句号、问号、逗号等停顿处断开
- 任务执行：服务端内存任务队列，适合本地开发和原型验证
- 每段生成：统一使用 `asset://<assetId>` 作为参考图
- Prompt 控制：每段统一复用固定人设模板和固定口播镜头模板
- 时长控制：每个分段可单独设置 4-15 秒，服务端按分段秒数分别调用模型
- 查询方式：前端轮询本地任务，服务端轮询 Seedance 异步任务

## 已知限制

- 当前任务状态保存在内存中，重启开发服务器后会丢失
- 未实现失败片段单独重试
- 未接入视频结果自动转存，Seedance 原始视频 URL 通常只有短期有效期，生产环境建议转存到 TOS 或 R2
- 片段之间不做尾帧续接，当前版本优先保证清晰度和人物稳定性
